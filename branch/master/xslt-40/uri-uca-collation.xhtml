1 Introduction

This document describes the URI http://www.w3.org/2013/collation/UCA.

This URI defines a family of collations. It does not define a namespace; therefore it is not associated with a namespace prefix or with an XML Schema. For updated information, please refer to the latest version of the [XSLT 3.0] specification (section 13.4, The Unicode Collation Algorithm).

Collation URIs are used throughout the XSLT, XPath, and XQuery specifications for operations such as comparing and sorting strings. The set of collation URIs that may be used for this purpose is open-ended. XSLT 3.0 defines a family of collation URIs corresponding to different parameterizations of the Unicode Collation Algorithm: see [UCA]. XSLT 3.0 processors are required to support this family of collations; XSLT 2.0 (and XPath 2.0/3.0 and XQuery 1.0/3.0) processors may also support them, insofar as the set of collation URIs accepted by such processors is implementation-defined.

A specific collation within the family of collations is selected by appending query parameters to the URI. For example, the URI http://www.w3.org/2013/collation/UCA?lang=de selects a collation suitable for use with German text.

The Unicode Collation Algorithm must not be confused with Unicode Codepoint Collation.

The XSLT specification allows use of different versions of the Unicode Collation Algorithm and encourages use of the latest version. The collation URIs are not expected to change in response to changes in the Unicode specification.

This document specifies the use of URIs to invoke and parameterize the Unicode Collation Algorithm. The [XSLT 3.0] and [XPath and XQuery Functions and Operators 3.1] specifications are the only specifications that are permitted to amend this family of collation URIs.

2 Collations

The Unicode Collation Algorithm is a family of collations for sorting strings in different languages and according to different publishing conventions. The interpretation of this URI is described (with identical text) in the XSLT 3.0 specification, and in the XPath/XQuery Functions and Operators 3.1 specification.

Each host language defines its own conformance requirements.
